## 我们是谁 | Who We Are

**PRISM Agentic** 致力于构建下一代自进化多智能体系统。我们相信 AI Agent 不应该是一次性的工具，而应该是能够持续学习、自主进化的数字团队成员。

**PRISM Agentic** builds the next generation of self-evolving multi-agent systems. We believe AI agents should not be disposable tools — they should be digital team members that continuously learn and autonomously evolve.

---

## 核心理念 | Core Philosophy

```
传统 Agent：每次从零开始，重复犯同样的错误
PRISM Agent：每次任务都积累经验，每个项目都让整个团队更强

Traditional Agent: Starts from scratch every time, repeats the same mistakes
PRISM Agent: Accumulates experience from every task, every project makes the whole team stronger
```

我们围绕三个核心信念构建 PRISM：

**1. 活的智能体（Living Agents）**
每个 Agent 拥有固定身份和成长记忆。不是静态提示词，而是会随项目经验持续进化的数字专家。

**2. 自适应编排（Adaptive Orchestration）**
不是所有项目都需要同样的流程。PRISM Flow 根据项目规模和风险自动选择最合适的执行模式。

**3. 原生进化循环（Native Evolution Loop）**
自我改进不是事后叠加的功能，而是从第一天就融入每个 Agent、每次任务、每个交接的核心机制。

## 技术栈 | Tech Stack

- **智能体规范**：YAML Frontmatter + Markdown（人类可读，机器可解析）
- **编排协议**：PRISM Flow（质量门控 + 证据驱动 + 上下文连续性）
- **进化引擎**：Python + SQLite/PostgreSQL + OpenAI API
- **集成协议**：MCP（Model Context Protocol）

## 为什么选择 PRISM？| Why PRISM?

| 对比维度 | 传统 Agent 框架 | PRISM |
|---------|---------------|-------|
| 经验积累 | 无，每次从零开始 | 跨会话持久记忆 |
| 质量保障 | 依赖人工检查 | 自动化质量门控 + Critic Agent |
| 自我改进 | 需要人工重写提示词 | 三级自动进化（L1/L2/L3） |
| 团队协作 | Agent 之间无感知 | 显式协作图谱 + 结构化交接 |
| 漂移检测 | 无 | 自动性能监控 + 告警 |
